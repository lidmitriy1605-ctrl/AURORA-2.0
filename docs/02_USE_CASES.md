# 02. Пользовательские сценарии AURORA 2.0

## Назначение

Документ описывает первые реальные сценарии. Они определяют MVP; функция без сценария не добавляется.

## UC-01. Сохранить личную заметку

Дмитрий сообщает системе факт, идею или решение. AURORA сохраняет запись в его личной области, указывает дату и источник, затем возвращает идентификатор записи.

**Готово, когда:** заметка не видна другому пользователю и её можно найти по запросу.

## UC-02. Найти сохранённую информацию

Пользователь запрашивает ранее сохранённую заметку. Система ищет только в разрешённой области данных и показывает найденные записи с датой и источником.

**Готово, когда:** поиск не раскрывает личные данные другого пользователя.

## UC-03. Вести личную задачу

Пользователь создаёт задачу, меняет её статус и просматривает открытые задачи. Каждое изменение фиксируется в журнале.

**Готово, когда:** задача имеет владельца, статус и историю изменений.

## UC-04. Работать с семейной задачей

Дмитрий или Ева создают задачу в семейной области. Оба пользователя могут видеть её и работать с ней в рамках семейного контура.

**Готово, когда:** семейная задача доступна обоим, но не смешивается с личными задачами.

## UC-05. Запросить внешнее действие

Пользователь просит выполнить действие во внешнем сервисе. AURORA подготавливает действие, показывает последствия и выполняет его только после явного подтверждения.

**Готово, когда:** без подтверждения внешнее действие не выполняется.

## Не входит в MVP

Голос, мобильное приложение, Telegram, почта, календарь, автоматические агенты и подключение реальных внешних сервисов. Они добавляются только после проверки базовых сценариев.
