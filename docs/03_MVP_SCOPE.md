# 03. Границы MVP AURORA 2.0

## Цель MVP

Дать Дмитрию работающий, локальный и проверяемый контур для заметок и задач с разделением личной и семейной информации.

## Входит

- пользователи Дмитрий и Ева;
- области `dmitry`, `eva`, `family`;
- создание и поиск заметок;
- создание, просмотр и изменение статуса задач;
- журнал существенных действий;
- механизм подготовки и подтверждения критических действий;
- локальное хранилище и интерфейс командной строки;
- автоматические тесты ключевых правил изоляции.

## Не входит

- реальные внешние интеграции и отправка данных;
- вход по паролю, облачная синхронизация и многопользовательский сетевой доступ;
- автоматическое выполнение внешних действий;
- LLM, голосовой, веб-, мобильный или Telegram-интерфейс.

## Критерии готовности

1. Личные данные одного пользователя нельзя прочитать через область другого.
2. Семейные данные доступны обоим семейным пользователям.
3. Изменение задачи и запрос критического действия журналируются.
4. Неподтверждённое критическое действие не выполняется.
5. Проверки запускаются одной командой на чистом Python.
