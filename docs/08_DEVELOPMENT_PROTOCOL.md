# 08. Протокол разработки

## Рабочий цикл

1. Сверить `PROJECT_STATE.md`, dashboard и Git.
2. Выбрать один небольшой результат из текущего milestone.
3. Реализовать его вместе с тестами и документацией.
4. Запустить проверки.
5. Обновить state, dashboard, session log и changelog.
6. Создать понятный Git-коммит и синхронизировать с GitHub.

## Границы автономии

Исполнитель может самостоятельно менять код, тесты, документацию, структуру проекта и внутренние автоматизации в рамках утверждённого MVP.

Отдельное подтверждение Дмитрия требуется для удаления данных, подключения внешних учётных записей, передачи личных данных за пределы устройства, расходов, изменения правил доступа и выпуска в публичный доступ.

## Definition of Done

Результат считается завершённым, если он реализован, проверен, описан, отражён в management-файлах и зафиксирован в Git.
